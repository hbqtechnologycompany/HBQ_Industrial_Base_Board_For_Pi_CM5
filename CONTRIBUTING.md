# Contributing to CM5 Gateway

Thank you for your interest in contributing! We welcome issues, discussions, docs, and code improvements.

## How to Contribute
1. Fork the repo and create a feature branch from `main`
2. Make focused changes with clear commit messages
3. Add/update README or docs when behavior changes
4. Open a Pull Request describing the change, motivation, and testing

## Branching and PRs
- Branch naming: `feat/…`, `fix/…`, `docs/…`, `chore/…`
- Keep PRs small and scoped; link related issues
- Include reproduction steps or logs for bug fixes

## Coding Guidelines
- Prefer clarity over cleverness
- Avoid catching errors without handling
- Match existing formatting; do not reformat unrelated code

## Issues
- Use issue templates (bug/feature)
- Provide OS/board revision, module type, logs, and steps to reproduce

## Security
- Do not disclose security issues publicly. See `SECURITY.md` to report privately.

## Recognition
Contributors will be credited in release notes when applicable.
